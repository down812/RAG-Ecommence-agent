# 页面重构与接口对齐清单

本清单用于约束本轮 UI 重构范围：只做现有后端接口、现有前端状态或可由现有数据计算支撑的能力；没有接口支撑的功能不做真实交互。

## 结论总览

| 页面 | 重构结论 | 说明 |
| --- | --- | --- |
| 首页 | 可做 | 使用商品分页、搜索、商品详情、加购物车等现有能力 |
| AI 导购 | 可做 | 使用流式聊天、图片附件、停止生成、评价、会话接口 |
| AI 咨询历史 | 可做 | 使用会话列表、会话详情、删除会话接口 |
| 商品详情 | 可做 | 使用商品详情、SKU、价格、库存、图片、加购物车接口 |
| 购物车 | 可做 | 使用购物车获取、数量更新、删除、清空接口 |
| 我的/管理中心 | 可做 | 使用本地登录态、用户类型与现有管理模块入口 |
| 商品管理 | 可做 | 使用商品列表、新增、编辑、删除、上下架、图片上传接口 |
| 用户管理 | 可做 | 使用子账号列表、新增、编辑、删除接口与后端权限校验 |
| 知识库管理 | 可做 | 使用数据集列表、新增、启停、删除、上传、文件列表、下载、删除接口 |
| 评价管理 | 部分可做 | 可做评价列表、评分筛选、删除、前端统计；不做客服回复 |

## 页面明细

### 1. 首页

可做：
- 顶部搜索入口。
- AI 导购入口和快捷需求标签。
- 商品分类入口。
- 推荐商品列表。
- 商品详情跳转。
- 加入购物车。
- 底部导航。

接口/数据来源：
- `GET /api/product/list/page`
- `GET /api/product/search`
- `GET /api/product/detail/{id}`
- `POST /api/cart/add`

降级规则：
- “为你推荐”先使用商品分页或搜索结果组织展示，不做真正个性化推荐接口。
- 活动 Banner、学生优惠等若没有后端配置，作为静态 UI 或先不做。

### 2. AI 导购

可做：
- 文本对话。
- 图片附件。
- 流式生成。
- 停止生成。
- AI 推荐商品卡。
- 查看商品详情。
- 加入购物车。
- 对 AI 回复点赞/点踩。
- 历史会话入口。

接口/数据来源：
- `POST /ai/chat/messages`
- `POST /ai/chat/messages/{sessionId}/{messageId}/stop`
- `POST /evaluate/add`
- `POST /api/cart/add`
- AI 流式结果中的结构化商品推荐。

降级规则：
- 商品卡内容以 AI 返回结构化结果和商品 DTO 现有字段为准。
- 若 AI 返回没有完整商品字段，则只展示可用字段，不补造库存、评分等信息。

### 3. AI 咨询历史

可做：
- 历史会话列表。
- 继续对话。
- 删除会话。
- 按现有会话标题/摘要展示。

接口/数据来源：
- `GET /ai/chat/sessions`
- `GET /ai/chat/sessions/{sessionId}`
- `DELETE /ai/chat/sessions/{sessionId}`

降级规则：
- 分类筛选如“手机/电脑/耳机”没有独立接口，可先做前端关键词过滤，或暂不做。
- “推荐商品数量”仅在会话摘要数据可得时展示。

### 4. 商品详情

可做：
- 商品主图。
- 商品标题、价格、原价。
- SKU/规格标签。
- 核心参数。
- 加入购物车。
- 查看购物车。
- 用户评价入口的摘要展示。

接口/数据来源：
- `GET /api/product/detail/{id}`
- `POST /api/cart/add`
- `GET /api/cart/get`
- 商品 DTO 字段：`title`、`brand`、`category`、`basePrice`、`mainImageUrl`、`skuList`、`stock`、`attributeList`。

降级规则：
- “AI 推荐理由”只在从 AI 推荐结果进入或上下文有推荐理由时展示；普通商品入口不强行展示。
- 评分、好评率、评价标签如没有商品维度评价接口，只做弱化展示或隐藏。
- 收藏功能没有接口，暂不做真实收藏。

### 5. 购物车

可做：
- 购物车商品列表。
- 数量加减。
- 删除单项。
- 清空购物车。
- 总价统计。
- 去结算入口。

接口/数据来源：
- `GET /api/cart/get`
- `POST /api/cart/add`
- `PUT /api/cart/update`
- `DELETE /api/cart/remove/{cartItemId}`
- `DELETE /api/cart/clear`

降级规则：
- 后端没有订单/支付接口；结算页与成功页只能沿用现有前端本地流程。
- “来自 AI 推荐”没有购物车字段，不能作为真实后端状态；可隐藏或作为本地临时提示。
- 单项勾选/全选如当前数据层没有选择状态，只作为前端 UI 状态处理，不传后端。

### 6. 我的/管理中心

可做：
- 当前用户信息展示。
- 管理员身份标签。
- 管理中心入口。
- AI 咨询历史入口。
- 购物车入口。
- 退出登录。
- 底部导航。

接口/数据来源：
- 本地 `TokenProvider` 中的 `userId`、`userType`、`identifier`。
- `POST /auth/logout`

降级规则：
- 通知、权限详情页若无接口，先不做真实内容。
- 用户头像如无字段，使用默认头像或统一插画。

### 7. 商品管理

可做：
- 商品列表。
- 搜索/筛选状态。
- 新增商品。
- 编辑商品。
- 删除商品。
- 上架/下架。
- 上传/替换商品图片。
- SKU 编辑。
- 前端统计：商品总数、上架数、下架数。

接口/数据来源：
- `GET /api/product/list`
- `GET /api/product/list/page`
- `GET /api/product/search`
- `POST /api/product/create`
- `PUT /api/product/update`
- `DELETE /api/product/delete/{id}`
- `POST /api/product/upload/image`
- `PUT /api/product/update/image`

降级规则：
- “待审核”没有明确状态枚举时不做，或只在后端状态存在时展示。
- 库存预警可由 SKU `stock` 前端计算，但阈值需要前端约定。

### 8. 用户管理

可做：
- 用户列表。
- 新增用户。
- 编辑密码/角色。
- 删除用户。
- 按角色筛选。
- 前端统计：用户总数、管理员数、普通用户数。

接口/数据来源：
- `POST /user/subaccountList`
- `POST /user/subaccount`
- `PUT /user/subaccount/{subaccountId}`
- `DELETE /user/subaccount/{subaccountId}`

降级规则：
- “禁用/启用用户”没有独立字段或接口，暂不做真实禁用。
- 手机号、头像、昵称如果 DTO 没有字段，不在列表中硬补。
- 删除权限由后端控制，普通管理员不可删除时前端也应隐藏或禁用。

### 9. 知识库管理

可做：
- 知识库列表。
- 新增知识库。
- 启用/禁用知识库。
- 删除知识库。
- 上传文件。
- 查看文件列表。
- 下载/打开文件。
- 删除文件。
- 前端统计：知识库数量、文件数量、总大小。

接口/数据来源：
- `GET /ai/dataset/list`
- `POST /ai/dataset/create`
- `POST /ai/dataset/toggle/{datasetId}`
- `DELETE /ai/dataset/{datasetId}`
- `POST /ai/dataset/upload/{datasetId}`
- `GET /ai/dataset/{datasetId}/files`
- `GET /ai/dataset/file/{fileId}`
- `DELETE /ai/dataset/file/{fileId}`

降级规则：
- “最新更新时间”若数据集没有更新时间字段，可用文件 `createdAt` 的最大值估算，或隐藏。
- 文档总大小由文件列表 `fileSize` 前端汇总。

### 10. 评价管理

可做：
- 评价列表。
- 按评分筛选。
- 删除评价。
- 查看评论文本、会话 ID、消息 ID、用户 ID、创建时间。
- 前端统计：总数、有帮助数、没帮助数。

接口/数据来源：
- `GET /evaluate/page`
- `DELETE /evaluate/delete/{id}`
- `POST /evaluate/add`
- `PUT /evaluate/update`

暂不做：
- 客服回复。
- 已回复/待回复真实状态。
- 回复时间。
- 导出评价数据。
- 商品维度综合评分，除非后端补充对应聚合接口。

降级规则：
- 截图中的星级评价与当前接口的 `rating=1/-1` 不完全一致，应按现有“有帮助/没帮助”模型重构样式。

## 暂不纳入本轮真实功能

- 真实订单创建、订单列表、订单详情。
- 支付、支付结果查询、退款。
- 收藏。
- 客服回复评价。
- 用户禁用/启用。
- 后台数据导出。
- 服务端个性化推荐接口。
- 独立统计看板接口。

## 实施约束

- 页面样式向十张图靠齐，交互能力以本清单为边界。
- 可以做前端计算统计，但不能伪装成服务端真实统计。
- 无接口支撑的按钮应隐藏、禁用，或降级为静态信息。
- 涉及删除、清空、角色变更的操作必须保留确认弹窗。
- 空状态、加载中、错误状态需要统一样式，但不改变接口行为。
